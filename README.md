## 🕹️ LEGACY STACK :: THE RETRO TECH 🕹️

### 💾 WHAT WE'RE DEALING WITH 💾

| Component | Technology | Year | Vibe |
|-----------|-----------|------|------|
| **Language** | VBScript 5.0 | 1998 | 📟 Beeper era |
| **Framework** | Classic ASP 3.0 | 2000 | 💿 Y2K compliant! |
| **Data Access** | ADO 2.8 (ActiveX Data Objects) | 2000 | 🎲 Inline SQL everywhere |
| **Database** | SQL Server 2012 Express | 2012 | 🗄️ Still kicking |
| **Authentication** | Session variables | 1996 | 🔓 What's a JWT? |
| **Email** | CDO (Collaboration Data Objects) | 1998 | 📬 SMTP hardcoded |
| **File Upload** | FileSystemObject | 1996 | 💾 No validation gang |
| **Web Server** | IIS 6/7 | 2003 | 🌐 IE6 optimized |
| **Layout** | `<table>` tags | 1993 | 🎨 Spacer GIFs unite! |
| **Code Reuse** | Server Side Includes (SSI) | 1995 | 📂 `<!--#include file="header.asp"-->` |

### 🚨 ANTI-PATTERNS DETECTED 🚨

- ❌ **SQL Injection** — String concatenation in queries: `"SELECT * FROM Users WHERE Username='" & username & "'"`
- ❌ **Plain-text Passwords** — Stored directly in database, no hashing
- ❌ **No CSRF Protection** — Session-based auth with zero token validation
- ❌ **Inline SQL** — Every ASP page has ADO recordsets with raw SQL
- ❌ **On Error Resume Next** — Global error suppression (production chaos mode)
- ❌ **Hardcoded Credentials** — SMTP, database connections in includes/conn.asp
- ❌ **No Input Validation** — QueryString parameters used raw
- ❌ **FileSystemObject Uploads** — No size limits, no type checking
- ❌ **Table Layouts** — Nested tables for days, zero responsiveness
- ❌ **Mixed Concerns** — HTML, VBScript, SQL all on the same page

**🎪 IT'S A SECURITY CIRCUS, AND EVERYONE'S INVITED! 🎪**
